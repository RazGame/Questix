## Роли И Команды

Нужно переделать модель ролей и команд.

### Администратор

- Модерирует все игры.
- Назначает роли другим пользователям.

### Организатор

- Создаёт игры.
- Модерирует только игры, которые создал сам.

### Капитан Команды

- Создаёт команду.
- Приглашает участников в команду.
- Управляет составом своей команды.
- Может быть капитаном только одной команды.
- Может передать права капитана любому участнику своей команды.

Команда должна хранить:

- название;
- список участников;
- капитана.

### Участник

- Редактирует свой профиль.
- Принимает участие в команде.
- Может выйти из команды.

## Командное Прохождение

В игре участвует команда, а не отдельный пользователь.

- Все участники команды могут вводить ответы.
- Если любой участник команды ввёл правильный ответ, вся команда переходит на следующее задание.
- Прогресс прохождения должен быть привязан к команде и к заявке команды на игру.
- Индивидуальный пользователь не должен иметь отдельный игровой прогресс внутри командного квеста.

## Логи Команды

Система должна логировать все действия команды во время игры.

Нужно сохранять:

- кто из участников отправил ответ;
- текст отправленного ответа;
- правильный или неправильный ответ;
- время отправки ответа;
- задание, к которому относится ответ;
- переход команды на следующее задание;
- старт и завершение прохождения.

Доступ к логам:

- администратор видит логи всех игр;
- организатор видит логи только игр, которые создал сам.

## Публикация Результатов

Результаты игры должны быть доступны участникам только после того, как организатор завершит игру.

До завершения игры:

- участники не видят итоговые результаты других команд;
- участники не видят общую статистику прохождения;
- организатор и администратор могут смотреть технический прогресс и логи.

После завершения игры:

- участники игры могут посмотреть итоговую таблицу;
- участники игры могут посмотреть статистику прохождения по заданиям;
- результаты становятся read-only.

## Статистика После Игры

После завершения игры участникам должна быть доступна статистика прохождения в табличном виде.

Нужны представления:

- таблица команд по заданиям, где строки - команды, а колонки - задания;
- в каждой ячейке видно, какое задание команда проходила на этом шаге, кто отправил правильный ответ, дату и время прохождения, время на задание;
- отдельная колонка с итоговым временем команды;
- возможность сортировать таблицу по моменту прохождения заданий;
- возможность сортировать таблицу по времени, потраченному на задания;
- визуальное выделение итогового времени и места команды;
- отображение названия игры и общего контекста статистики.

Статистика должна помогать сравнить:

- общий маршрут команд;
- порядок прохождения заданий;
- скорость прохождения каждого задания;
- итоговое время;
- вклад участников команды через отправленные ответы.
